How WebBlock Works

WebBlock edits the Windows hosts file in C:/Windows/System32/drivers/etc to make a website, instead of routing to its IP/web server, route to 127.0.0.1 (localhost). This will show a 127.0.0.1 page instead of the address expected. Usually, 127.0.0.1 has nothing on it. If the computer user is a developer, it may have some developing page open. However, it is not possible for the website to be accessed from that page because any time any browser, iframe, or get request accesses the website, it will return localhost.

Please note that WebBlock's actions can be undone if the user has administrator privileges. However, it is highly unlikely that they know about this file, let alone how to use it.

If you would like to view the currently blocked sites, WebBlock V1.5 allows you to see all of the blocked entries. Please note: the Show Blocked Sites button shows both entries, the site and the www.site entry. This is to ensure that the user cannot connect to the site with using www.sitename.tld.

Is WebBlock traceable? Also, Will Deleting WebBlock Delete My Previous Blocking Entries?

WebBlock is not traceable! This means that you can download it, use it, delete it, and there will be no trace except for the hosts file changes. If you delete WebBlock your hosts file will NOT be returned to its original state! Please note that if you are deleting WebBlock to undo website blocking, this will do nothing and you should either use the 'Unblock' feature or navigate and change the entries yourself.
